01 Top-N per group

Get the top N rows within each group (top 3 products per category, etc).

When: "Top X for each Y" — the most common analytics question.
WITH ranked AS (
  SELECT name, category, price,
    DENSE_RANK() OVER (PARTITION BY category ORDER BY price DESC) AS rnk
  FROM products
)
SELECT * FROM ranked WHERE rnk <= 3;

Use ROW_NUMBER for exactly N (ties broken arbitrarily) · RANK for ties skipping · DENSE_RANK for distinct positions.

02 Running total

Cumulative sum that grows over time.

When: account balances, sales-to-date, leaderboard scores.
SELECT order_id, order_date, total_amount,
  SUM(total_amount) OVER (ORDER BY order_date) AS running_total
FROM orders;

Per-group running total: add PARTITION BY group_col.

03 Moving average (N-day)

Smooth out noisy time series.

When: stock prices, traffic graphs, smoothing seasonality.
SELECT order_date, total_amount,
  AVG(total_amount) OVER (
    ORDER BY order_date
    ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
  ) AS ma_7day
FROM orders;

ROWS BETWEEN N PRECEDING AND CURRENT ROW gives an N+1 row trailing window.

04 Latest row per group

Most recent order per customer, etc.

When: "current state per X" — last login, most recent transaction.
WITH ranked AS (
  SELECT *,
    ROW_NUMBER() OVER (PARTITION BY customer_id ORDER BY order_date DESC) AS rn
  FROM orders
)
SELECT * FROM ranked WHERE rn = 1;

05 Find duplicates

Detect rows that shouldn't exist twice.

When: data quality checks, dedupe planning.
SELECT email, COUNT(*) AS dup_count
FROM customers
GROUP BY email
HAVING COUNT(*) > 1;

06 Delete duplicates (keep one)

De-duplicate, retaining the row with the lowest ID.

When: data cleanup — always run inside a transaction.
DELETE FROM customers
WHERE customer_id NOT IN (
  SELECT MIN(customer_id) FROM customers GROUP BY email
);

⚠️ Always preview with SELECT first. Wrap in BEGIN ... COMMIT.

07 Pivot rows to columns

Transform long format to wide format (no PIVOT needed).

When: dashboards expecting one column per category.
SELECT customer_id,
  SUM(CASE WHEN status = 'delivered' THEN 1 ELSE 0 END) AS delivered,
  SUM(CASE WHEN status = 'pending' THEN 1 ELSE 0 END) AS pending,
  SUM(CASE WHEN status = 'cancelled' THEN 1 ELSE 0 END) AS cancelled
FROM orders GROUP BY customer_id;

Conditional aggregation works on every database, unlike vendor-specific PIVOT syntax.

08 Unpivot columns to rows

Reverse of pivot — wide to long.

When: normalizing a denormalized export, prepping data for BI.
SELECT id, 'q1' AS quarter, q1_sales AS sales FROM sales_wide
UNION ALL
SELECT id, 'q2', q2_sales FROM sales_wide
UNION ALL
SELECT id, 'q3', q3_sales FROM sales_wide
UNION ALL
SELECT id, 'q4', q4_sales FROM sales_wide;

09 Find gaps in a sequence

Missing IDs, missing dates.

When: auditing data integrity, finding missing invoice numbers.
SELECT id + 1 AS missing_id_start,
  LEAD(id) OVER (ORDER BY id) - 1 AS missing_id_end
FROM invoices
WHERE LEAD(id) OVER (ORDER BY id) - id > 1;

10 Gaps and islands (consecutive runs)

Group consecutive rows with the same value or consecutive dates.

When: longest streaks, consecutive wins, sessions.
WITH grp AS (
  SELECT user_id, login_date,
    login_date - ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY login_date) * INTERVAL '1 day' AS streak_id
  FROM logins
)
SELECT user_id, MIN(login_date) AS start_date, MAX(login_date) AS end_date, COUNT(*) AS streak_length
FROM grp GROUP BY user_id, streak_id;

11 Sessionization (group events by inactivity gap)

Bucket events into sessions when gap > N minutes.

When: product analytics, user behavior funnels.
WITH gapped AS (
  SELECT user_id, ts,
    CASE WHEN ts - LAG(ts) OVER (PARTITION BY user_id ORDER BY ts) > INTERVAL '30 minutes'
         THEN 1 ELSE 0 END AS new_session
  FROM events
)
SELECT user_id, ts,
  SUM(new_session) OVER (PARTITION BY user_id ORDER BY ts) AS session_id
FROM gapped;

12 Cohort retention

% of users from cohort X still active in month X+N.

When: startup metrics, product analytics, growth dashboards.
WITH cohorts AS (
  SELECT user_id, DATE_TRUNC('month', signup_date) AS cohort_month
  FROM users
)
SELECT c.cohort_month,
  DATE_PART('month', AGE(a.activity_month, c.cohort_month)) AS months_since,
  COUNT(DISTINCT a.user_id) AS active_users
FROM cohorts c
JOIN activity a USING (user_id)
GROUP BY c.cohort_month, months_since;

13 Percentile / quartile

Find the 75th percentile salary.

When: statistical analysis, removing outliers.
-- PostgreSQL / Oracle
SELECT PERCENTILE_CONT(0.75) WITHIN GROUP (ORDER BY salary) AS p75
FROM employees;

-- Portable using NTILE
SELECT MAX(salary) FROM (
  SELECT salary, NTILE(4) OVER (ORDER BY salary) AS q
  FROM employees
) t WHERE q = 3;

14 Median (no MEDIAN function)

Portable median computation.

When: robust to outliers, salary reporting, etc.
WITH ranked AS (
  SELECT salary,
    ROW_NUMBER() OVER (ORDER BY salary) AS rn,
    COUNT(*) OVER () AS cnt
  FROM employees
)
SELECT AVG(salary) AS median
FROM ranked
WHERE rn IN ((cnt + 1) / 2, (cnt + 2) / 2);

15 Month-over-month growth %

Compare each month to the previous.

When: executive dashboards, board decks, finance reports.
WITH monthly AS (
  SELECT DATE_TRUNC('month', order_date) AS month, SUM(total_amount) AS rev
  FROM orders GROUP BY DATE_TRUNC('month', order_date)
)
SELECT month, rev,
  LAG(rev) OVER (ORDER BY month) AS prev_rev,
  ROUND(100.0 * (rev - LAG(rev) OVER (ORDER BY month)) / LAG(rev) OVER (ORDER BY month), 2) AS growth_pct
FROM monthly;

16 Year-to-date (YTD)

Cumulative within the current year.

When: finance reports, performance tracking.
SELECT order_date, total_amount,
  SUM(total_amount) OVER (
    PARTITION BY EXTRACT(YEAR FROM order_date)
    ORDER BY order_date
  ) AS ytd
FROM orders;

17 Fill missing dates (calendar table)

Zero out days with no activity.

When: time-series charts that need every day shown.
-- PostgreSQL
WITH cal AS (
  SELECT generate_series(CURRENT_DATE - INTERVAL '29 days', CURRENT_DATE, INTERVAL '1 day')::DATE AS day
)
SELECT cal.day, COALESCE(SUM(o.total_amount), 0) AS revenue
FROM cal LEFT JOIN orders o ON o.order_date = cal.day
GROUP BY cal.day ORDER BY cal.day;

18 Random sample

N random rows.

When: spot-check large tables, build sampled training sets.
-- PG / SQLite
SELECT * FROM orders ORDER BY RANDOM() LIMIT 100;

-- MySQL
SELECT * FROM orders ORDER BY RAND() LIMIT 100;

-- PG: faster on huge tables (TABLESAMPLE)
SELECT * FROM orders TABLESAMPLE BERNOULLI (1);  -- ~1% sample

19 Anti-join (NOT EXISTS)

Find rows in A with no matching row in B.

When: "customers who haven't ordered", "products never sold", etc.
SELECT * FROM customers c
WHERE NOT EXISTS (
  SELECT 1 FROM orders o WHERE o.customer_id = c.customer_id
);

Faster and NULL-safer than NOT IN.

20 Recursive hierarchy (org chart)

Traverse tree structures: managers → reports → reports' reports.

When: org charts, category trees, comment threads.
WITH RECURSIVE org AS (
  -- Anchor: top-level managers
  SELECT employee_id, name, manager_id, 1 AS level
  FROM employees WHERE manager_id IS NULL

  UNION ALL

  -- Recursive: each level reporting to the previous
  SELECT e.employee_id, e.name, e.manager_id, org.level + 1
  FROM employees e
  JOIN org ON e.manager_id = org.employee_id
)
SELECT * FROM org ORDER BY level;

21 Upsert (INSERT or UPDATE)

Insert a row, but if it already exists, update it instead.

When: sync jobs, idempotent ingestion, settings tables.
-- PostgreSQL
INSERT INTO products (product_id, name, price)
VALUES (101, 'New Mouse', 29.99)
ON CONFLICT (product_id) DO UPDATE SET
  name = EXCLUDED.name, price = EXCLUDED.price;

-- MySQL
INSERT INTO products (product_id, name, price)
VALUES (101, 'New Mouse', 29.99)
ON DUPLICATE KEY UPDATE name = VALUES(name), price = VALUES(price);

-- SQLite
INSERT INTO products (product_id, name, price) VALUES (101, 'New Mouse', 29.99)
ON CONFLICT(product_id) DO UPDATE SET name = excluded.name;

22 Conditional aggregation

Count/sum subsets in one pass.

When: KPI dashboards with breakdowns.
SELECT
  COUNT(*) AS total_orders,
  SUM(CASE WHEN status = 'delivered' THEN 1 ELSE 0 END) AS delivered_count,
  SUM(CASE WHEN status = 'delivered' THEN total_amount ELSE 0 END) AS delivered_revenue,
  AVG(CASE WHEN total_amount > 100 THEN total_amount END) AS avg_high_value
FROM orders;

23 String aggregation

Concat all values in a group into one string.

When: reporting tags, comma-separated lists in outputs.
-- PostgreSQL
SELECT customer_id,
  STRING_AGG(CAST(order_id AS TEXT), ', ' ORDER BY order_date) AS all_orders
FROM orders GROUP BY customer_id;

-- MySQL
SELECT customer_id, GROUP_CONCAT(order_id ORDER BY order_date SEPARATOR ', ') FROM orders GROUP BY customer_id;

-- SQLite
SELECT customer_id, GROUP_CONCAT(order_id, ', ') FROM orders GROUP BY customer_id;

24 Difference from previous row (LAG)

Compute deltas between consecutive rows.

When: price changes, sensor data deltas, time-series.
SELECT order_date, total_amount,
  LAG(total_amount) OVER (ORDER BY order_date) AS prev,
  total_amount - LAG(total_amount) OVER (ORDER BY order_date) AS change
FROM orders;

25 Bulk insert from query

Copy filtered rows into another table.

When: archiving, materializing reports, ETL.
-- Insert filtered subset
INSERT INTO orders_archive (order_id, customer_id, order_date, total_amount)
SELECT order_id, customer_id, order_date, total_amount
FROM orders
WHERE order_date < CURRENT_DATE - INTERVAL '1 year';

-- Create+populate in one step
CREATE TABLE top_customers AS
SELECT customer_id, SUM(total_amount) AS ltv
FROM orders GROUP BY customer_id HAVING SUM(total_amount) > 1000;
