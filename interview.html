📚 Interview prep meta-tips

⚡ The 80/20 of SQL interviews

  • JOINs — 90% of all questions involve at least one JOIN
  • GROUP BY + aggregates — every "total per X" question
  • Window functions — RANK, ROW_NUMBER, LAG, running totals
  • NULL behavior — the #1 gotcha set
  • Top-N per group — asked in some form by everyone

🎯 How to answer in interviews

  1. Restate the question in your own words
  2. Ask about edge cases — NULLs, ties, empty tables
  3. Talk through the approach before coding
  4. Write the query, explaining each clause
  5. Mention alternatives — "we could also use a CTE here"
  6. Discuss performance — what indexes would help?

⚠️ Mistakes that fail interviews

  • Jumping straight to writing without clarifying
  • Using SELECT * when specific columns are needed
  • Forgetting NULL handling
  • Using = NULL instead of IS NULL
  • WHERE with aggregates (should be HAVING)
  • Not knowing the difference between RANK/DENSE_RANK/ROW_NUMBER
  • No idea about indexes when asked "how would you optimize?"
  • Silence — talk through your thinking

🎤 What SQL rounds actually look like

PHONE SCREEN · 30 MIN

The warmup

2-3 medium questions. They want to see you can write a JOIN with a GROUP BY without freezing. Pattern matches everything in our Level 2-3.

Common companies: TCS, Infosys, mid-size SaaS, most startups

ONSITE · 45-60 MIN

The deep dive

1 hard problem (window functions, gaps/islands, retention) with multiple follow-ups. They want to see how you think under pressure and iterate. Level 4 territory.

Common companies: Google, Meta, Amazon, Stripe, Airbnb, Netflix

SYSTEM DESIGN · 60 MIN

The schema design

"Design a schema for X" + queries against it. Tests normalization, indexing intuition, trade-offs (OLTP vs OLAP). Level 5 + interview prep here.

Common companies: Senior+ roles at most FAANG, data engineering roles
