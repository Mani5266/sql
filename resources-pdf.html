01 The Honest Roadmap

Don't grind every site. Follow this exact sequence and you'll be interview-ready in 4-6 weeks.

📍 The path that actually works

1
Finish SQLSENSEI Levels 1-3 first
Get the fundamentals locked in (SELECT, JOIN, GROUP BY, subqueries). Don't skip the exercises.
2
Grind LeetCode SQL Easy → Medium (~50 problems)
This builds the muscle memory. Aim for 2-3 problems/day for 3-4 weeks.
3
Switch to DataLemur / StrataScratch for company-tagged questions
Filter by your target company for the last 2-3 weeks before interviews. Real questions, real format.
4
Do the 8 Week SQL Challenge (optional but powerful)
If you want to feel what real analytics work looks like. Builds confidence and portfolio.
5
Mock interview on Pramp before the real thing
Free peer-to-peer mock interviews. Talking through SQL out loud is a separate skill.

02 Tier 1 — Start Here

These three sites cover 80% of what you need. If you only use these three, you'll be fine.

LeetCode SQL
leetcode.com/problemset/database/
FREEMIUM ★ BEST OVERALL EASY → HARD FAANG
250+ SQL problems tagged by difficulty. The de facto standard for SQL interview prep. Discussion forums for every problem.
Why: Every FAANG interviewer has seen these problems. Solving the Top 50 SQL list essentially guarantees you've seen the pattern before in your interview.
DataLemur
datalemur.com
FREEMIUM ★ BEST FOR FAANG DS META · AMAZON · NETFLIX
Made by an ex-Meta data scientist. Real interview questions from Meta, Amazon, Netflix, etc. Clean UI, focused purely on SQL for data roles.
Why: The free questions are higher quality than most paid platforms. Specifically targets data science / analytics SQL rounds.
StrataScratch
stratascratch.com
FREEMIUM ★ MOST COMPANIES 50+ COMPANIES
Real questions from 50+ companies with company tags, difficulty filters, and detailed solutions. Strong free tier.
Why: Widest variety of company-specific questions. If you're targeting a less common company (Uber, Airbnb, Stripe), this beats LeetCode.

03 For Fundamentals (Beginners)

If you're still learning basics — use these alongside SQLSENSEI Levels 1-2.

SQLZoo
sqlzoo.net
FREE BEGINNER
Interactive lessons, no signup. Goes from SELECT to window functions step by step.
When: Perfect right after SQLSENSEI Level 1-2 to reinforce basics with different practice problems.
Mode SQL Tutorial
mode.com/sql-tutorial/
FREE BEGINNER → INTERMEDIATE
Well-written tutorials using realistic startup datasets. Best free SQL course on the internet for quality of writing.
Why: Teaches SQL the way analysts actually use it, not the way textbooks do.
SQLBolt
sqlbolt.com
FREE ABSOLUTE BEGINNER
Bite-size interactive lessons, very gentle. Great if you're completely new and SQLZoo feels too fast.
When: Day 1-3 of your SQL journey. Read a chapter, do the exercises, done.
W3Schools SQL Tutorial
w3schools.com/sql/
FREE REFERENCE
Syntax reference with "Try it yourself" sandboxes. Not a course, but a useful lookup for syntax.
When: Use it like a dictionary, not a learning path.

Advent of SQL
adventofsql.com
FREE ESCALATING
Yearly puzzle event in December. One SQL puzzle per day for 25 days, getting progressively harder.
When: December — great way to keep your skills sharp with a community.
Select Star SQL
selectstarsql.com
FREE INTERMEDIATE

🎯 The Honest Recommendation

Skip 90% of these. More platforms ≠ more skill. Here's exactly what to do:

Week 1-2: Finish SQLSENSEI Levels 1-3.

Week 3-5: LeetCode SQL — 50 Easy + 30 Medium problems. That's it.

Week 6 (pre-interview): DataLemur, filter by your target company. Do 15-20 questions.

Week 7+: Pramp for 2-3 mock interviews. Talk through your SQL out loud.

Don't: Bounce between platforms thinking variety = mastery. Pick one, finish it, then move on.

08 Pro Tips for Practice

⚡ The 80/20 of SQL

  • JOINs — in 90% of questions
  • GROUP BY + aggregates
  • Window functions (RANK, ROW_NUMBER, LAG)
  • NULL behavior (#1 gotcha)
  • Top-N per group

🎤 How to answer in interviews

  • Restate the question in your words
  • Ask about NULLs, ties, edge cases
  • Talk through approach before coding
  • Mention alternatives (CTE vs subquery)
  • Discuss performance + indexes

🛠 Practice routine

  • 2-3 problems per day, not 10
  • 30 min max per problem — then peek
  • Type queries by hand, no copy-paste
  • Re-solve hard ones 1 week later
  • Write a one-line comment explaining the trick

⚠️ Mistakes that fail interviews

  • Jumping to code without clarifying
  • Using = NULL not IS NULL
  • WHERE with aggregates (use HAVING)
  • Confusing RANK / ROW_NUMBER / DENSE_RANK
  • Silence — talk through your thinking
